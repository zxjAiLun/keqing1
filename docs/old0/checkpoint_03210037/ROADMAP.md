# 后续计划

## 当前状态

往返转换已修复（1948/1948 匹配），数据处理管道畅通。

```
数据源 ──→ tenhou6 ──→ mjai ──→ 训练 ──→ 模型 ──→ Bot
                      ↓
                   tenhou6 (验证)
```

## 短期计划 (1-2周)

### 1. 数据质量提升

**目标**: 清洗和扩展高质量训练数据

- [ ] 扩大数据集规模（当前 ds3 约 2000 局）
  - 目标：至少 10,000 局
  - 来源：天凤凤凰场/王座场高段位牌谱
- [ ] 过滤低质量数据
  - 去除 AI对战牌谱
  - 去除异常结局（极端分数、流局）
- [ ] 数据增强
  - 镜像翻转（同色牌互换）
  - 玩家视角变换

### 2. 模型训练优化

**目标**: 提升模型性能

- [ ] 超参数调优

- [ ] 训练策略

- [ ] 评估指标完善

### 3. 特征工程

## 资源需求

### 计算资源

| 阶段 | GPU | CPU | 存储 |
|------|-----|-----|------|
| 数据处理 | 1x RTX 3080 | 8核 | 100GB |
| 小型训练 | 1x RTX 3080 | 8核 | 100GB |
| 中型训练 | 1x A100 | 16核 | 500GB |
| 大型训练 | 4x A100 | 32核 | 1TB |

### 数据资源

- 天凤牌谱（约 100GB）
- 雀魂牌谱（约 50GB）
- 人工标注数据（视需求）

## 里程碑

| 阶段 | 目标 | 指标 |
|------|------|------|
| M1 | 基础可用 | 胜率 > 60% vs RuleBot |
| M2 | 良好体验 | 胜率 > 70%，和牌率 > 25% |
| M3 | 高手水平 | 稳定凤东，胜率 > 75% |
| M4 | 超越人类 | 与顶尖人类对战胜率 > 50% |

## 风险与挑战

1. **数据质量**: 牌谱来源复杂，需大量清洗
2. **计算资源**: 大模型训练耗时长、成本高
3. **评估难度**: 麻将顺位波动大，需长期评估
4. **环境变化**: 天凤/雀魂协议可能变更
