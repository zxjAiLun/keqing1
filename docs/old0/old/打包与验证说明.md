# 打包与验证说明（给接手同学）

本文说明当前项目如何把最新模型+代码打包为 `mjai` 可运行的 submission zip，并验证包内行为是否正确。

## 1. 打包目的

- 生成一个 zip，供 `mjai.Simulator` / Docker 环境加载运行。
- 保证 zip 内使用的是**最新源码**和**最新权重**，避免“本地改了但包里没更新”。

## 2. 打包内容（必须包含）

- `artifacts/mjai_submission/bot.py`
  - 入口脚本，模拟器会执行它。
- `src/`
  - 主要逻辑代码，尤其是 `src/bot/mjai_bot.py`。
- `checkpoints/best.npz`
  - 训练得到的模型参数。
- `checkpoints/best.json`
  - 无 numpy 依赖版本（Docker 推理更稳）。

## 3. 关键代码在哪

- 打包入口脚本：`artifacts/mjai_submission/bot.py`
- 核心决策逻辑：`src/bot/mjai_bot.py`
- 导出 json 权重脚本：`src/tools/export_checkpoint_json.py`
- 自战+导出 tenhou6：`src/tools/run_mjai_simulator_and_export_tenhou6.py`
- 打包前后行为一致性检查：`src/tools/check_bundle_parity.py`

## 4. 标准执行命令（按顺序）

以下假设当前模型目录是 `artifacts/sl_v2_ds3_torch_001`。

### 步骤 A：导出 `best.json`

```bash
source .venv/bin/activate && PYTHONPATH=src python3 -m tools.export_checkpoint_json \
  --checkpoint artifacts/sl_v2_ds3_torch_001/best.npz \
  --out artifacts/sl_v2_ds3_torch_001/best.json
```

### 步骤 B：重新打包 zip（包含最新源码）

```bash
python3 - <<'PY'
import shutil, zipfile
from pathlib import Path

root = Path('/home/bailan/project/keqing1')
build = Path('/tmp/mjai_bundle_build_latest')
if build.exists():
    shutil.rmtree(build)
build.mkdir(parents=True)

shutil.copy2(root / 'artifacts/mjai_submission/bot.py', build / 'bot.py')
shutil.copytree(root / 'src', build / 'src')

(build / 'checkpoints').mkdir(parents=True, exist_ok=True)
shutil.copy2(root / 'artifacts/sl_v2_ds3_torch_001/best.npz', build / 'checkpoints/best.npz')
shutil.copy2(root / 'artifacts/sl_v2_ds3_torch_001/best.json', build / 'checkpoints/best.json')

out = root / 'artifacts/mjai_submission/bot_bundle_v2_ds3_latest.zip'
if out.exists():
    out.unlink()

with zipfile.ZipFile(out, 'w', compression=zipfile.ZIP_DEFLATED) as z:
    for p in build.rglob('*'):
        if p.is_file():
            z.write(p, p.relative_to(build))

print(out)
PY
```

### 步骤 C：检查“包内外行为一致”

```bash
source .venv/bin/activate && PYTHONPATH=src python3 -m tools.check_bundle_parity \
  --bundle-zip artifacts/mjai_submission/bot_bundle_v2_ds3_latest.zip \
  --log-path dataset/example/mjai_log_example.jsonl \
  --player-id 0 \
  --checkpoint artifacts/sl_v2_ds3_torch_001/best.npz \
  --out artifacts/debug/bundle_parity_v2_ds3_latest.json
```

### 步骤 D：跑一次自战并导出 tenhou6

```bash
source .venv/bin/activate && PYTHONPATH=src python3 -m tools.run_mjai_simulator_and_export_tenhou6 \
  --submission-zip artifacts/mjai_submission/bot_bundle_v2_ds3_latest.zip \
  --logs-dir artifacts/mjai_sim_logs/selfplay_v2_ds3_latest_001 \
  --output-json dataset/bot_export/mjai_selfplay_v2_ds3_latest_001.json \
  --timeout 10 \
  --seed-start 0 \
  --seed-end 3
```

## 5. 如何判断是否“打包成功且用的是新逻辑”

- `check_bundle_parity` 输出 `ok=true`。
- 自战日志目录中存在：
  - `mjai_log.json`
  - `policy_debug_summary.json`
- tenhou6 导出文件存在：
  - `dataset/bot_export/mjai_selfplay_v2_ds3_latest_001.json`

## 6. 常见坑

- 只更新了本地 `src/`，但没重新打包 zip。
- 只拷贝了 `best.npz`，没导出/拷贝 `best.json`。
- 用旧 zip 路径启动模拟器，导致误以为“修改没生效”。
